Document Structure
web-html.html — Head 
<!-- Every HTML file starts with this declaration -->
<!DOCTYPE html>
<html lang="en">

  <!-- Character encoding — always UTF-8 -->

  <meta name="viewport" content="width=device-width, initial-scale=1.0">

01
The <head> — invisible setup
The <head> is never shown on screen — it's the page's settings panel. charset="UTF-8" lets you use any language character (Arabic, French, German…). The viewport meta tag is what makes your page look good on a phone instead of zoomed out. link rel="stylesheet" tells the browser to load your CSS file and apply its styles.

JavaScript
web-html.html — Script 
<script>
  // 1. Cursor glow — follows mouse position
  const glow = document.getElementById('cursorGlow');
  document.addEventListener('mousemove', e => {
    glow.style.left = e.clientX + 'px';
    glow.style.top  = e.clientY + 'px';
  });

  // 2. Scroll-to-top button — shows after scrolling 400px
  const scrollBtn = document.getElementById('scrollTopBtn');
  window.addEventListener('scroll', () => {
    scrollBtn.classList.toggle('visible', window.scrollY > 400);
  });

  // 3. Mobile menu toggle — adds/removes "open" class
  const toggle   = document.getElementById('navToggle');
  const navLinks = document.getElementById('navLinks');
  toggle.addEventListener('click', () => {
    navLinks.classList.toggle('open');
  });

  // 4. Scroll reveal — watches cards and animates them in
  const io = new IntersectionObserver(entries => {
    entries.forEach(e => {
      if (e.isIntersecting) e.target.classList.add('revealed');
    });
  }, { threshold: 0.1 });

  document.querySelectorAll('.scard, .flip-wrap')
    .forEach(el => io.observe(el));
</script>
04
JavaScript — 4 small interactions
Cursor glow: Every time the mouse moves, we update the position of a div using style.left and style.top. The glow effect is just CSS blur on that div.

Scroll button: classList.toggle('visible', condition) — adds the class if condition is true, removes it if false. One line instead of an if/else.

Scroll reveal: IntersectionObserver watches elements and fires when they enter the viewport. We just add 'revealed' which triggers the CSS fade-in animation.

  box-sizing: border-box;
  margin: 0;
  padding: 0;

  color: var(--text);

01
CSS Variables — the color system
:root is the top of the document — variables defined here are available everywhere. var(--cyan) is how you use them. The huge advantage: if you want to change the accent color, you change it in ONE place and it updates everywhere automatically. The * reset removes default browser margins — without this, every browser renders spacing differently and your layout breaks.

Flexbox + sticky + glass blur

backdrop-filter: blur(20px) creates the frosted glass look — it blurs whatever is behind the element. The background needs to be semi-transparent (rgba(..., 0.85)) for this to work.

transition on the button means any property change (opacity, position) animates smoothly instead of jumping instantly.

03
@keyframes — how CSS animations work
@keyframes defines what an animation does. You name it, then describe the states at 0% (start) and 100% (end), with optional steps in between.

animation: float 4s ease-in-out infinite — name, duration, timing curve, repeat count. ease-in-out makes it start slow, go fast in the middle, end slow — which is why the avatar looks like it's floating naturally instead of moving robotically.

The scroll reveal trick: cards start opacity: 0 and shifted down. JavaScript adds .revealed → CSS transitions them to opacity: 1 and position 0. The transition property makes that change animate smoothly.

04
Media queries — making it work on phones
@media (max-width: 768px) means "apply these styles ONLY when the screen is 768px wide or narrower". Everything inside overrides the default desktop styles.

The mobile menu works by: hiding .nav-links with display: none by default, then JavaScript adds the .open class when the hamburger is clicked, and the CSS .nav-links.open rule shows it again with display: flex. No extra libraries needed — pure HTML + CSS + 3 lines of JS.

Hero layout — full screen with floating avatar
min-height: calc(100vh - 68px) — 100vh = 100% of the screen height. We subtract the navbar height (68px) so the hero fills exactly the visible area.

Flip Card CSS
web-css.css — Flip Card 
/* The outer card — sets the 3D perspective depth */
.flip-card {
  width: min(720px, 100%);  /* max 720px, never exceeds screen */
  height: 380px;
  perspective: 1200px;    /* distance of viewer from the card */
}

/* The inner — this is what actually rotates */
.flip-inner {
  position: relative;
  width: 100%; height: 100%;
  transform-style: preserve-3d; /* children exist in 3D space */
  transition: transform 0.7s ease;
}

/* Hover triggers the rotation */
.flip-card:hover .flip-inner {
  transform: rotateY(180deg); /* spins 180° around Y axis */
}

/* Both faces stack on top of each other */
.flip-front, .flip-back {
  position: absolute;
  inset: 0;
  backface-visibility: hidden; /* hides the face when rotated away */
  border-radius: 22px;
}

/* Back face starts pre-rotated so it appears correct after flip */
.flip-back {
  transform: rotateY(180deg);
  background: var(--bg-card);
  display: flex;
  align-items: center;
  justify-content: center;
}
06
3D flip card — how it actually works
The trick uses three CSS properties together:

perspective on the parent creates the 3D depth. Higher value = flatter/more subtle. Lower value = more dramatic distortion.

transform-style: preserve-3d on the inner container tells the browser its children live in 3D space, not flat on the page.

backface-visibility: hidden on both faces hides them when they're pointing away from you — without this you'd see the back face mirrored through the front.

The back face starts at rotateY(180deg) — pre-rotated so when the card flips another 180°, it arrives upright. The whole animation is just one transition on the transform property.

}
07
CSS Grid + pseudo-elements + hover effects
grid-template-columns: repeat(2, 1fr) — repeat(2, ...) means "do this 2 times", 1fr means "1 fraction of available space". Both columns are equal width automatically.

grid-column: 1 / -1 on the German card — -1 means "the last column line". So it stretches from the first line to the last line = full width.

::before is a pseudo-element — a fake element inserted before the card's content using only CSS, no extra HTML needed. We use it for the cyan top-border that sweeps in on hover. scaleX(0) makes it 0 width, scaleX(1) makes it full width — the transition animates between them.
